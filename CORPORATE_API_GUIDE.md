# CORPORATE API KEY SETUP
# Настройка корпоративного ключа для ITSM365 Sales Trainer

## 🏢 Зачем нужен корпоративный ключ?

### Проблема с индивидуальными ключами:

Когда каждый сотрудник использует свой личный Groq API ключ:
- ❌ Нет контроля над расходами
- ❌ Нельзя отследить кто использует
- ❌ Сложно управлять доступом
- ❌ Риски безопасности (ключи в браузерах)

### Решение — корпоративный ключ:

✅ Централизованное управление
✅ Контроль расходов и лимитов
✅ Мониторинг использования
✅ Безопасность — один ключ для команды
✅ Возможность отозвать доступ

## 🚀 Настройка корпоративного ключа

### Шаг 1: Создайте корпоративный аккаунт Groq

1. Перейдите на [console.groq.com](https://console.groq.com)
2. Создайте корпоративный аккаунт используя рабочую почту:
   - **Рекомендуется:** `it@yourcompany.com` или `api@yourcompany.com`
   - **Не используйте:** личные почты сотрудников

3. Подтвердите email и завершите регистрацию

### Шаг 2: Создайте API ключ

1. В меню слева выберите **"API Keys"**
2. Нажмите **"Create API Key"**
3. Заполните форму:
   - **Name:** `ITSM365 Sales Trainer - Corporate`
   - **Description:** `Corporate key for sales team training`
4. Скопируйте созданный ключ

> **ВАЖНО:** Сохраните ключ в безопасном месте! Он будет показан только один раз.

### Шаг 3: Настройте лимиты (опционально)

Groq предоставляет бесплатно 10,000 запросов/час. Для corp аккаунта можно настроить:

1. В меню выберите **"Settings"** → **"Rate Limits"**
2. Настройте лимиты если нужно:
   - Requests per hour: 10,000 (максимум для бесплатного tier)
   - Requests per day: неограничено
   - Requests per month: неограничено

### Шаг 4: Распространите ключ команде

Есть три способа:

#### Вариант A: Через код (для IT команды)

**Измените файл `js/app.js`:**

```javascript
// Найдите строку:
this.ai = new AIManager({
    provider: 'groq',
    apiKey: localStorage.getItem('itsm365_api_key') || null
});

// Замените на:
this.ai = new AIManager({
    provider: 'groq',
    apiKey: 'gsk_your_corporate_key_here' // <-- Вставьте корпоративный ключ
});
```

**Пересоберите и задеплойте приложение:**
```bash
git add js/app.js
git commit -m "Add corporate API key"
git push
```

#### Вариант B: Через переменную окружения (для продвинутых)

Измените `js/ai.js` чтобы поддерживать env variables:

```javascript
constructor(config = {}) {
    this.apiKey = config.apiKey ||
                  window.ITSM365_API_KEY || // Глобальная переменная
                  null;
    // ...
}
```

В `index.html` добавьте:
```html
<script>
    window.ITSM365_API_KEY = 'gsk_your_corporate_key_here';
</script>
<script src="js/app.js"></script>
```

#### Вариант C: Для разработки (тестирование)

Создайте файл `config.js` в папке `js/`:

```javascript
// js/config.js
window.ITSM365_CONFIG = {
    GROQ_API_KEY: 'gsk_your_corporate_key_here'
};
```

В `index.html` добавьте перед app.js:
```html
<script src="js/config.js"></script>
<script src="js/app.js"></script>
```

## 🔒 Безопасность

### Правила корпоративного ключа:

✅ **ДО:**
- Используйте корпоративную почту для регистрации
- Храните ключ в безопасном месте (password manager)
- Ограничьте доступ к ключу
- Проверяйте использование регулярно
- Отзывайте ключ при необходимости

❌ **НЕ:**
- Не публикуйте ключ в открытом репозитории (Git)
- Не отправляйте ключ по email в открытом виде
- Не делитесь ключом с внешними лицами
- Не используйте личные аккаунты сотрудников

### Защита ключа в коде:

Если вы ключ в коде (Вариант A), обязательно:

1. **Добавьте в `.gitignore`:**
```
# API Keys
js/app.js
js/config.js
```

2. **Создайте `.env.example` для референса:**
```
# Corporate API Key (DO NOT commit actual key)
ITSM365_API_KEY=gsk_your_corporate_key_here
```

3. **Используйте переменные окружения на хостинге:**
- Netlify: Environment Variables в настройках сайта
- Vercel: Environment Variables в настройках проекта

## 📊 Мониторинг использования

### Groq Console:

1. В меню выберите **"Usage"** → **"API Usage"**
2. Вы увидите:
   - Requests per hour/day/month
   - Response time
   - Error rate

### Оценка расхода:

**Пример:**
- Команда: 20 менеджеров
- Тренировок в день: 3 на человека
- Сообщений в тренировке: ~10
- Итого: 20 × 3 × 10 = **600 запросов/день**

**Лимит бесплатного tier:** 10,000 requests/hour = **240,000 requests/day**

> 💡 **Вывод:** Бесплатного лимита Groq хватает для команды любого размера!

## 🔄 Обновление и отмена ключа

### Обновить ключ:

1. В Groq Console → **"API Keys"**
2. Создайте новый ключ
3. Обновите код/конфигурацию
4. Задеплойте
5. Старый ключ можно отозвать

### Отозвать ключ:

1. В Groq Console → **"API Keys"**
2. Найдите корпоративный ключ
3. Нажмите **"Revoke"**
4. Подтвердите

> Ключ перестанет работать немедленно после отзыва.
