# ITSM365 Тренажёр для обучения продаж

Интерактивный веб-тренажёр для отдела продаж ITSM365. Позволяет тренироваться проводить демонстрации, работать с возражениями и закрывать сделки с виртуальным клиентом.

![ITSM365 Sales Trainer](https://img.shields.io/badge/ITSM365-Sales%20Trainer-blue)
![License](https://img.shields.io/badge/license-MIT-green)
![Free](https://img.shields.io/badge/price-FREE-brightgreen)

## 🎯 Возможности

- **Голосовое взаимодействие** - говорите с виртуальным клиентом голосом
- **5 встроенных сценариев** - демо, возражения, закрытие сделки
- **3 уровня сложности** - новичок, средний, продвинутый
- **Оценка результатов** - AI анализирует диалог и даёт рекомендации
- **Прогресс тренировок** - отслеживайте улучшение результатов
- **Бесплатный хостинг** - работает на Netlify/GitHub Pages
- **Без ограничений** - бесплатный доступ для всех сотрудников

## 🚀 Быстрый старт

### 1. Получить бесплатный API ключ

Тренажёр использует **Groq API** для генерации ответов виртуального клиента.

1. Перейдите на [console.groq.com](https://console.groq.com)
2. Зарегистрируйтесь (бесплатно)
3. Создайте новый API ключ в разделе Keys
4. Сохраните ключ - он понадобится при первом запуске

> **Groq API бесплатно предоставляет 10,000 запросов в час** - этого достаточно для интенсивного использования.

### 2. Запустить приложение

#### Вариант A: Локально (для разработки)

```bash
# Клонировать репозиторий
git clone https://github.com/your-org/itsm365-sales-trainer.git
cd itsm365-sales-trainer

# Открыть index.html в браузере
open index.html  # macOS
start index.html # Windows
xdg-open index.html # Linux
```

Или использовать локальный сервер:

```bash
# Python 3
python -m http.server 8000

# Node.js (если установлен http-server)
npx http-server
```

Откройте `http://localhost:8000` в браузере.

#### Вариант B: Деплой на Netlify (рекомендуется)

1. Загрузите файлы проекта в GitHub репозиторий
2. Перейдите на [netlify.com](https://netlify.com)
3. Нажмите "Add new site" → "Import from Git"
4. Выберите ваш репозиторий
5. Готово! Получите ссылку для команды

### 3. Начать обучение

1. Откройте приложение по ссылке
2. Выберите сценарий из списка
3. Нажмите "Начать сценарий"
4. Введите ваш Groq API ключ (только один раз, сохранится в браузере)
5. Начните общаться с виртуальным клиентом!

## 🎤 Голосовое взаимодействие

Тренажёр поддерживает голосовой ввод и вывод:

### Голосовой ввод
- Нажмите кнопку 🎤 в поле ввода сообщения
- Разрешите доступ к микрофону (требуется для первого раза)
- Говорите чётко, текст появится автоматически
- Сообщение отправится после завершения фразы

### Голосовой вывод
- Виртуальный клиент отвечает голосом
- Работает автоматически после каждого сообщения
- Использует синтез речи браузера (бесплатно)

### Поддержка браузеров

| Firefox | ❌ Нет         | ✅ Полностью    |
| Edge    | ✅ Полностью   | ✅ Полностью    |

> **Рекомендуется Chrome** для полной функциональности.

## 🔧 Добавление новых сценариев

Сценарии описаны в файле `js/app.js` в методах с префиксом `get`. Для добавления нового сценария:

1. Найдите секцию "СЦЕНАРИИ" в `app.js`
2. Добавьте новый метод по аналогии:

```javascript
getMyNewScenario() {
    return {
        id: 'my-new-scenario',
        category: 'demo', // demo, objections, closing
        difficulty: 'intermediate', // beginner, intermediate, advanced
        duration: 15,
        title: 'Название сценария',
        description: 'Описание сценария',
        client_profile: {
            name: 'Имя клиента',
            role: 'Должность',
            company: 'Компания',
            industry: 'Отрасль',
            pain_points: [
                'Боль 1',
                'Боль 2'
            ],
            greeting: 'Приветствие клиента'
        },
        objectives: [
            'Цель 1',
            'Цель 2'
        ],
        system_prompt: 'Инструкция для AI как играть роль клиента...',
        success_criteria: {
            criteria_1: 'Описание критерия 1',
            criteria_2: 'Описание критерия 2'
        }
    };
}
```

3. Добавьте сценарий в массив в методе `loadScenarios()`:

```javascript
loadScenarios() {
    return [
        this.getBankingScenario(),
        this.getTelecomScenario(),
        // ... другие сценарии
        this.getMyNewScenario() // <-- ваш новый сценарий
    ];
}
```

## 🔐 Безопасность

- API ключ сохраняется **только в localStorage браузера**
- Никогда не отправляется на сторонние серверы
- Каждый пользователь использует свой собственный ключ
- Для корпоративного использования рекомендуется настроить корпоративный Groq ключ

### Корпоративный API ключ

Для команд рекомендуем использовать **один корпоративный ключ** вместо индивидуальных:

**Преимущества:**
- ✅ Централизованное управление
- ✅ Контроль расходов и лимитов
- ✅ Мониторинг использования
- ✅ Безопасность — один ключ для команды
- ✅ Возможность отозвать доступ

**Как настроить:** см. [CORPORATE_API_GUIDE.md](CORPORATE_API_GUIDE.md)

**Кратко:**
1. Создайте корпоративный аккаунт на [console.groq.com](https://console.groq.com)
2. Создайте API ключ
3. Замените в `js/app.js` строку:
```javascript
apiKey: localStorage.getItem('itsm365_api_key') || null
```
на:
```javascript
apiKey: 'gsk_your_corporate_key_here'
```
4. Задеплойте обновление

### Groq Free Tier Limits

- **10,000 запросов/час** — бесплатно
- **240,000 запросов/день** — максимум на бесплатном тарифе
- Хватает для команды любого размера!

**Пример расчёта:**
- Команда: 20 менеджеров
- Тренировок в день: 3 на человека
- Сообщений в тренировке: ~10
- **Итого:** 600 запросов/день (в пределах лимита)

## ⚙️ Конфигурация

### Смена AI провайдера

По умолчанию используется Groq API. Для смены провайдера:

1. Откройте `js/ai.js`
2. Измените провайдер при инициализации в `js/app.js`:

```javascript
this.ai = new AIManager({
```

## 🐛 Решение проблем

### "Голосовой ввод недоступен"

- Используйте браузер Chrome для полной функциональности
- Проверьте разрешение на использование микрофона в настройках браузера

### "Не удалось подключиться к API"

- Проверьте что API ключ введён правильно
- Убедитесь что ключ активен в console.groq.com
- Проверьте подключение к интернету

### "AI отвечает медленно"

- Попробуйте сменить модель в конфигурации
- Проверьте скорость интернета
- Groq обычно отвечает за 1-2 секунды

### "Оценка диалога недоступна"

- Проверьте что диалог не слишком короткий (минимум 3 сообщения)
- Попробуйте завершить сценарий заново
